# @cpi-crm/importer

Безопасный воспроизводимый импорт исходной XLSX-книги. Формулы не вычисляются:
в raw staging сохраняется только выражение формулы, cached result игнорируется.
СНИЛС и даты рождения редактируются до staging: сохраняются координаты ячейки,
заголовок и hash строки, но не исходное значение. Все остальные ячейки сохраняются
для field-level provenance и просмотра пользователем с `imports.read_raw`.

```bash
pnpm --filter @cpi-crm/importer run import -- --dry-run \
  --file ../../Участники_всех_мероприятий_Стартап_студии_ЯДРО1.xlsx
```

Commit требует существующие организацию и пользователя. Если существующий
`file_objects.id` не передан, создаётся идемпотентная запись в bucket
`local-import` по SHA-256 файла.

```bash
pnpm --filter @cpi-crm/importer run import -- --commit \
  --file ../../Участники_всех_мероприятий_Стартап_студии_ЯДРО1.xlsx \
  --database-url postgresql://cpi:cpi@localhost:5432/cpi_crm \
  --organization-id 00000000-0000-4000-8000-000000000000 \
  --user-id 00000000-0000-4000-8000-000000000001
```

Оба режима создают `import-report.json` и `import-report.md`. Dry-run не
открывает PostgreSQL. Commit блокируется, если любая контрольная сверка не
пройдена. Commit создаёт профили, контакты, мероприятия, факты участия и их
provenance. Ячейки с `pitch deck`, `презентация` или `материал` в заголовке
импортируются как отдельные артефакты, связанные с мероприятием и всеми
участниками строки. Неизвестная legacy-дата даёт доказанную активацию, но не
считается текущей активностью.

До создания рабочих сущностей name hygiene отклоняет только очевидно непригодные
ФИО: пустые/generated placeholders, явные test/abracadabra/qwerty-значения,
служебные маркеры, URL/email/телефон/число вместо имени и строки без достаточного
числа букв. Наблюдение получает `REJECTED`, полностью непригодная строка — `IGNORED`,
но immutable staging остаётся для аудита. После commit `AUTO_DEDUPE_V1`
транзакционно разрешает безопасные совпадения и оставляет журналируемый обратимый
merge. Повторный запуск той же книги идемпотентен.

Проверенный результат для штатной книги: 34 листа, 11 739 source rows, 208 330 raw
cells и 12 122 observations. Из них 12 110 разрешены, 12 отклонены, 11 source rows
проигнорированы. До дедупликации создаются 2 898 валидных профилей, 5 606 контактов,
68 мероприятий, 12 021 участие, 56 артефактов/версий и 118 contributor-связей.
После 1 602 решений кандидатов (`1252 MERGED / 5 NOT_DUPLICATE / 345 DISMISSED`)
остаётся 2 113 канонических участников и 0 `OPEN`. Повторный commit создаёт 0 новых
сущностей и решений.
