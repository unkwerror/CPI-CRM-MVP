# Риски и принятые допущения

1. В книге 135 вариантов заголовков и несколько человек в одной строке. Импортёр
   использует адаптеры листов и не объединяет карточки по одному ФИО.
2. `artifact_baseline_at` по умолчанию задан датой начала проекта, но перед пилотом
   администратор должен утвердить его явно.
3. Часовая зона по умолчанию — `Asia/Novosibirsk`; она хранится в настройках.
4. Локальные секреты и тестовые пароли пригодны только для разработки.
5. Значения чувствительных legacy-полей редактируются до raw staging; координаты,
   заголовки и hash строк остаются для аудита, сами значения не индексируются и не
   выводятся в интерфейсе участника.
6. Автоматическое объединение по одному ФИО или контакту создаёт риск ложного merge.
   Поэтому `AUTO_DEDUPE_V1` требует одновременно совместимые имена, чистый общий
   контакт и отсутствие конфликта external ID. Неоднозначные связи закрываются без
   объединения; каждое фактическое объединение журналируется и обратимо.
7. Фильтр ФИО намеренно консервативен: он отклоняет только явные test/placeholder,
   URL/email/телефон вместо имени, строки без букв и однозначную бессмыслицу. Редкие
   или однословные реальные имена не удаляются автоматически. Уже импортированная
   непригодная карточка архивируется с сохранением аудита, а не стирается физически.
8. Экспорт может содержать большой объём персональных данных. Он доступен только с
   `exports.bulk`, записывается в аудит, отдаётся потоково с `no-store` и никогда не
   восстанавливает 1 991 отредактированное чувствительное значение. Полученный файл
   должен храниться и передаваться по правилам организации.
9. Отдельные UI-разделы задач, артефактов и дублей убраны в пользу реестров участников
   и мероприятий. Служебные backend-таблицы и операции остаются для истории,
   целостности данных и безопасного отката.
10. Полный browser E2E OIDC/upload/antivirus зависит от Keycloak, MinIO и ClamAV.
    В текущем окружении Docker daemon недоступен, поэтому этот контур необходимо
    повторить перед серверным пилотом; остальные проверки выполняются отдельно.
