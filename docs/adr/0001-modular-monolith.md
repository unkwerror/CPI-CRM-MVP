# ADR-0001: модульный монолит для MVP

- Статус: принято
- Дата: 2026-07-22

## Контекст

Репозиторий был пустым, кроме ТЗ и исходной книги на 34 листа. Объём данных — около
2 тысяч канонических участников и 11 739 исходных строк. Нужно сохранить provenance,
неразрушающий merge и возможность позднее перенести локальную установку на сервер.

## Решение

Использовать TypeScript-монорепозиторий с Next.js, Fastify, отдельным worker и общими
пакетами домена/контрактов/БД. PostgreSQL является источником истины, Redis — только
транспортом фоновых задач, MinIO — приватным S3-совместимым хранилищем. Локальная
авторизация работает через настоящий OIDC-провайдер Keycloak.

Доменный расчёт lifecycle не зависит от фреймворков и использует внедряемый `Clock`.
Импорт — воспроизводимый ETL с immutable staging; строка Excel не равна участнику.

## Последствия

- серверное развёртывание меняет конфигурацию сервисов, а не доменную модель;
- локальный запуск требует Docker;
- микросервисы и Elasticsearch не вводятся до подтверждённой необходимости;
- импорт можно повторять и проверять без потери исходных строк.
